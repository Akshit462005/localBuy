üîß Technical Fix Details

Root Cause:

The CSS classes .btn.primary and .btn.secondary were defined incorrectly as .primary and .secondary, causing the button styling to not apply properly to elements with both btn and secondary classes.

CSS Changes Applied:

/* Fixed primary button styling */ .btn.primary { background-color: var(--primary-color); color: white; padding: 0.8rem 1.4rem; } .btn.primary:hover { background-color: #3a7bc8; transform: translateY(-2px); box-shadow: 0 8px 18px rgba(0,0,0,0.18); } /* Fixed secondary button styling */ .btn.secondary { background-color: #6c757d; color: white; border: 1px solid #6c757d; } .btn.secondary:hover { background-color: #5a6268; border-color: #545b62; transform: translateY(-2px); box-shadow: 0 6px 14px rgba(0,0,0,0.12); } /* Additional specific fix for quick buy button */ .quick-buy-btn { background-color: #6c757d !important; color: white !important; border: 1px solid #6c757d !important; font-weight: 600 !important; }
